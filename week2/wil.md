# WIL - 2주차: 훈련 세트와 테스트 세트, 데이터 전처리

이번 주차에서 가장 인상 깊었던 부분은 단순히 정확도 100%가 좋은 결과가 아닐 수 있다는 점이었다. 처음에 도미와 빙어 데이터를 KNN으로 분류했을 때 정확도가 100%로 나왔는데, 알고 보니 훈련 데이터와 테스트 데이터가 동일했기 때문이었다. 마치 시험 문제를 미리 알려주고 시험을 보는 것과 같다는 비유가 직관적으로 와닿았다.

훈련 세트와 테스트 세트를 분리했더니 이번엔 반대로 정확도가 0%가 나왔는데, 이 역시 샘플링 편향 때문이었다. 데이터를 앞에서부터 순서대로 자르다 보니 훈련 세트에는 도미만, 테스트 세트에는 빙어만 들어간 것이다. 이를 해결하기 위해 `numpy`로 인덱스를 섞거나 `train_test_split`의 `stratify` 파라미터를 활용해 타깃 비율을 유지하며 분리하는 방법을 배웠다.

또한 데이터 전처리의 필요성도 실감했다. 길이(cm)와 무게(g)는 단위와 범위가 완전히 달라서, KNN이 거리를 계산할 때 무게의 영향이 압도적으로 커지는 문제가 생겼다. 표준점수로 스케일을 맞춰주니 모델이 올바르게 도미를 분류했다. 직접 numpy 없이 평균, 분산, 표준편차를 수식으로 구현해보면서 표준화가 단순한 공식이 아니라 데이터의 분포를 균일하게 만드는 핵심 과정임을 이해할 수 있었다.
